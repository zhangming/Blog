    前端代码规范

    张明 2022/3/18

    # 赋值

    • 不会改变的值用 const
    • 会改变的用 let（尽量不适用 var，let 是块级作用域，而 var 是函数级作用域）块级作用域 https://es6.ruanyifeng.com/#docs/let
    • 不要使用八进制字面量，字符串字面量中也不要使用八进制转义字符
    //bad
const a = 0123
const copyright = 'Copyright \251'
//good
const a = '0123'

    # variables

    • 不要定义未使用的变量

    • 每个变量都用一个 const 或 let，const 放在一起，let 放在一起

    • 在你需要的地方声明变量，但是要放在合理的位置

    • 不要使用链式声明变量(链式声明变量会创建隐式全局变量)

    • 不要使用一元自增自减运算符

    • 在赋值的时候避免在 = 前/后换行

    • 不要对变量使用 delete 操作(https://juejin.cn/post/6844903918804172814)，采用设置成undefined的方式

    //bad
const a = 0123
//good
const a = '0123'

/ bad - 不必要的函数调用。
function checkName(hasName) {
  const name = getName();

  if (hasName === 'test') {
    return false;
  }

  if (name === 'test') {
    this.setName('');
    return false;
  }

  return name;
}

// good
function checkName(hasName) {
  if (hasName === 'test') {
    return false;
  }

  // 在需要的时候分配
  const name = getName();

  if (name === 'test') {
    this.setName('');
    return false;
  }

  return name;
}

// bad

  // JavaScript 将这一段解释为
  // let a = ( b = ( c = 1 ) );
  // let 只对变量 a 起作用; 变量 b 和 c 都变成了全局变量
  let a = b = c = 1;
}());
console.log(a); // undefined
console.log(b); // 1
console.log(c); // 1

// bad
const foo =
  superLongLongLongLongLongLongLongLongFunctionName();
// good
const foo = (
  superLongLongLongLongLongLongLongLongFunctionName()
);

    # 解构

    • 用对象的解构赋值来获取和使用对象某个或多个属性值

    • 用数组解构

    • 多个返回值用对象的解构，而不是数组解构。

    • 不要解构空值

    
// bad
function getFullName(user) {
  const firstName = user.firstName;
  const lastName = user.lastName;

  return `${firstName} ${lastName}`;
}

// good
function getFullName(user) {
  const { firstName, lastName } = user;
  return `${firstName} ${lastName}`;
}

// best
function getFullName({ firstName, lastName }) {
  return `${firstName} ${lastName}`;
}

const arr = [1, 2, 3, 4];

// bad
const first = arr[0];
const second = arr[1];

// good
const [first, second] = arr

    # 对象

    • 使用字面量（{} 是字面量，可以立即求值，而 new Object() 本质上是方法（只不过这个方法是内置的）调用，既然是方法调用，就涉及到在 proto 链中遍历该方法，当找到该方法后，又会生产方法调用必须的堆栈信息，方法调用结束后，还要释放该堆栈）

    • 用对象方法简写

    • 用属性值缩写（将你的所有缩写放在对象声明的开始）

    • 对象浅拷贝时，更推荐使用扩展运算符 ,获取对象属性时使用解构运算符

    • 不要对全局只读对象重新赋值

    • 外部变量不要与对象属性重名

    • 禁止使用 Object 构造器，禁止使用 Symbol 构造器，禁止使用 Function 构造器，禁止使用原始包装器(String)

    • getPrototypeOf 来替代 proto

    // bad
const atom = {
  value: 1,

  addValue: function (value) {
    return atom.value + value;
  },
};

// good
const atom = {
  value: 1,

  // 对象的方法
  addValue(value) {
    return atom.value + value;
  },
};

const lukeSkywalker = 'Luke Skywalker';

// bad
const obj = {
  lukeSkywalker: lukeSkywalker,
};

// good
const obj = {
  lukeSkywalker,
};
// good es6 扩展运算符 ...
const original = { a: 1, b: 2 };
// 浅拷贝
const copy = { ...original, c: 3 }; // copy => { a: 1, b: 2, c: 3 }

// rest 解构运算符
const { a, ...noA } = copy; // noA => { b: 2, c: 3 }

    # Array

    用字面量赋值

    • 用扩展运算符做数组浅拷贝

    • 用 ... 运算符而不是 Array.from 来将一个可迭代的对象转换成数组

    • 如果一个数组有很多行，在数组的 [ 后和 ] 前断行

    • 禁止使用稀疏数组

    //bad
var nums = new Array(1, 2, 3)
//good
var nums = [1, 2, 3]
// bad
const len = items.length;
const itemsCopy = [];
let i;
for (i = 0; i < len; i += 1) {
  itemsCopy[i] = items[i];
}
// good
const itemsCopy = [...items];
const arrLike = { 0: 'foo', 1: 'bar', 2: 'baz', length: 3 };
// bad
const arr = Array.prototype.slice.call(arrLike);
// good
const arr = Array.from(arrLike);

    # strings

    用单引号

    • 超过 100 个字符的字符串不应该用 string 串联成多行

    • 用字符串模板而不是字符串拼接来组织可编程字符串

    • 不使用 eval=>性能和安全性问题 https://www.zhihu.com/question/20591877

    # null&& undefined 的适用场景

    • 不要使用 undefined 来初始化变量

    • 避免空比较，对原始类型采用 typeof，对引用类型采用 instanceof，function 可以采用 typeof，属性采用 key in obj 判断，hasOwnProperty

     //bad
if (a == null) {
 //...
}

    # 函数

    • 用命名函数表达式而不是函数声明(在 ECMA-262 中 [块 block] 的定义是： 一系列的语句； 但是函数声明不是一个语句。 函数表达式是一个语句 http://www.adequatelygood.com/JavaScript-Scoping-and-Hoisting.html)

    • 把立即执行函数包裹在圆括号里(可以令其函数中声明的变量绕过 JavaScript 的变量置顶声明规则，还可以避免新的变量被解释成全局变量或函数名占用全局变量名的情况。与此同时它能在禁止访问函数内声明变量的情况下允许外部对函数的调用)

    • 不要用 arguments 命名参数，用 rest 语法...代替，避免使用 arguments.callee 和 arguments.caller

    • 不要丢掉异常处理中 err 参数，catch 中不要对错误重新赋值

    • 用默认参数语法而不是在函数里对参数重新赋值,不修改参数，把默认参数赋值放在最后(fixed)

    • 函数签名部分要有空格，括号于函数名

    • 调用或者书写一个包含多个参数的函数应该像其他多行代码写法一样： 每行只包含一个参数，每行逗号结尾

    • 点号操作符须与属性需在同一行

    • 函数调用时标识符与括号间不留间隔

    • 对象中定义了存值器，一定要对应的定义取值器

    • 使用 **dirname 和 **filename 时尽量避免使用字符串拼接(path.join)

    • return 语句中的赋值必需有括号包裹

        // bad
    function foo() {
      // ...
    }

// bad
    const foo = function () {
      // ...
    };

      // ...
    };

    # 箭头函数

    用函数表达式（在回调函数里）的时候就用箭头表达式

    如果函数体由一个没有副作用的表达式语句组成，删除大括号和 return。否则，继续用大括号和 return 语句

    万一表达式涉及多行，把他包裹在圆括号里更可读

    在隐式 return 中强制约束函数体的位置， 就写在箭头后面

    # 类

    不要扩展原生对象

    常用 class，避免直接操作 prototype

    构造函数以大写字母开头

    子类的构造器中一定要调用 super

    
Object.prototype.age = 123function Animal () {}var dog = new Animal()

    # Modules

    不要直接从 import 中直接 export 一个路径只 import 一次 多行 import 应该缩进，就像多行数组和对象字面量 Iterators & Generators 不要用遍历器。用 JavaScript 高级函数代替 for-in、 for-of 不使用 generator 而使用 async await

    # 运算符 & 等号

    （https://javascriptweblog.wordpress.com/2011/02/07/truth-equality-and-javascript/#more-2108） 用 === 和 !== 而不是 == 和 !=

    布尔值用缩写，而字符串和数字要明确比较对象

    在 case 和 default 分句里用大括号创建一块包含语法声明的区域

    三元表达式不应该嵌套，通常是单行表达式，避免不需要的三元表达式，对于三元运算符 ? 和 : 与他们所负责的代码处于同一行

    关系运算符的左值不要做取反操作

    Object.prototype.age = 123

const a = false
if(a) {
}

// good
switch (foo) {
  case 1: {
    let x = 1;
    break;
  }
  case 2: {
    const y = 2;
    break;
  }
}

if (!key in obj) {}

    # blocks

    单行代码块两边加空格 if 表达式的 else 和 if 的关闭大括号在一行 嵌套的代码块中禁止再定义函数

    function foo () { return true }

if (authenticated) {
  function setAuthUser () {}    // ✗ avoid
}

    # 控制语句

    当你的控制语句(if, while 等)太长或者超过最大长度限制的时候， 把每一个(组)判断条件放在单独一行里。 逻辑操作符放在行首 不要用选择操作符代替控制语句 控制语句后面要用 {},多行 if 语句的括号， 使用左花括号放在行尾的对齐方式，else 关键字要与花括号保持在同一行

    // good
if (
  (foo === 123 || bar === 'abc')
  && doesItLookGoodWhenItBecomesThatLong()
  && isThisReallyHappening()
) {
  thing1();
}

// bad
!isRunning && startRunning();

    # 注释

    多行注释用/**/,单行注释用//,注释前后面空格

    • 空格 ， 逗号，分号
    • 不要混合使用空格与制表符作为缩进
    • 行末不留空格
    • tab 用两个空格
    • 关键字后面加空格
    • 在大括号前空一格
    • 在控制语句(if, while 等)的圆括号前空一格。在函数调用和定义时，参数列表和函数名之间不空格
    • 用空格来隔开运算符
    • 文件结尾空一行
    • 当出现长的方法链（>2 个）时用缩进。用点开头强调该行是一个方法调用，而不是一个新的语句。
    • 在一个代码块后下一条语句前空一行
    • 不要用空白行填充块
    • 圆括号里不要加空格，方括号里不要加空格，花括号里加空格(模板字符串中变量前后不加空格)
    • 避免一行代码超过 100 个字符（包含空格）
    • , 前不要空格， , 后需要空格
    • 在对象的字面量属性中， 冒号后空格
    • (额外结尾逗号),不允许有多余的行末逗号,始终将逗号置于行末
    • 不要使用 (, [, or ` 等作为一行的开始.立即执行函数前加分号
    • 遇到分号时空格要后留前不留
    • 不允许有连续多行空行
    • 文件末尾留一空行
    for (let i = 0; i < items.length; i++) {...}

    # 命名

    • 用小驼峰式(keyValue)命名你的对象、函数、实例，用大驼峰式(KeyValue)命名类，【变量名义名词作为前缀，函数名用动词，尽量在变量名中体现出值的数据类型】

    • 不要用前置或后置下划线

    • 不要保存引用 this

    • export default 导出模块 A，则这个文件名也叫 A.*， import 时候的参数也叫 A。 大小写完全一致

    • 当你 export-default 一个函数时，函数名用小驼峰，文件名需要和函数名一致

    • 当你 export 一个结构体/类/单例/函数库/对象 时用大驼峰

    • 简称和缩写应该全部大写或全部小写

    # 其他

    • 将配置数据抽离
    • 单文件组件的文件名应该要么始终是单词大写开头 (PascalCase)，要么始终是横线连接 (kebab-case)。
    • 应用特定样式和约定的基础组件 (也就是展示类的、无逻辑的或无状态的组件) 应该全部以一个特定的前缀开头，比如 Base、App 或 V。
    • 只应该拥有单个活跃实例的组件应该以 The 前缀命名，以示其唯一性
    • 和父组件紧密耦合的子组件应该以父组件名作为前缀命名。(TodoList,TodolistItem,TodoListItemButton)
    • 组件名应该倾向于完整单词而不是缩写。
    • 在声明 prop 的时候，其命名应该始终使用 camelCase，而在模板和 JSX 中应该始终使用 kebab-case。
    • 组件模板应该只包含简单的表达式，复杂的表达式则应该重构为计算属性或方法
    • 应该把复杂计算属性分割为尽可能多的更简单的 property
    • 组件的实例顺序：name，components,directives,filters,mixins,props,data,computed,watch,methods
    • 如果一组 v-if + v-else 的元素类型相同，最好使用 key (比如两个<div> 元素)。

    # css

    • 将 css 从 javascript 中抽离
    • 在类名称中优先使用破折号而不是驼峰（class）
    • 不要使用 id 选择器
    • 在规则声明中使用多个选择器时，每个选择器占据一行
    • 在规则声明的左大括号{之前放置一个空格。
    • 在属性中，在：字符之后（但不能在该字符之前）放置一个空格
    • 将规则声明的右大括号}换行
    • 在规则声明之间放置空白行
    • 对于 border:使用 0 而不是 none 来指定样式没有边框
    • OOCSS 或“面向对象的 CSS”是一种写 CSS 的方法，它鼓励您将样式表视为“对象”的集合：可重复使用，可重复的代码段，可在整个网站中独立使用。
    • BEM“块元素修饰符”是 HTML 和 CSS 中类的命名约定
    • 顺序：布局属性（display / position / float / clear / visibility / overflow）=>自身属性（width / height / margin / padding / border / background）=>文本属性（color / font / text-decoration / text-align / vertical-align / white- space / break-word）=>其他属性 CSS3 浏览器私有前缀在前，标准前缀在后 vue 样式 为组件样式设置作用域（scoped 或 BEM 约定或 CSS Modules） vue dom v-for key 值 在 dom 模板中 my-component，JS/JSX 中的组件名应该始终是 PascalCase 的 元素 attribute 顺序：is, v-for，v-if (v-else-if v-show ),v-pre,v-once,id,ref,key,v-model,其他 attribute，v-on,v-html,v-text
